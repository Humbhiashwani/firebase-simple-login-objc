loginToFacebookAppWithId:permissions:audience:withCompletionBlock:

Attempts to log the user in to the Facebook app with the specified appId. The block will be called with the results of the attempt.

- (void)loginToFacebookAppWithId:(id)appId permissions:(id)permissions audience:(id)audience withCompletionBlock:(id)block

Parameters

appId

The Facebook application id for the app to log into. Make sure that the app has your bundle id registered in the facebook developer console

permissions

An array of strings, specifying the desired permissions for this user. If the array is empty, ‘email’ permission will be requested

audience

One of ACFacebookAudienceEveryone, ACFacebookAudienceFriends, ACFacebookAudienceOnlyMe, or nil. Required if your requested permissions include any write access. Assumed to be ACFacebookAudienceOnlyMe is nil is passed

block

A block that will be called with the results of the login attempt

Discussion

Attempts to log the user in to the Facebook app with the specified appId. The block will be called with the results of the attempt.

loginToTwitterAppWithId:multipleAccountsHandler:withCompletionBlock:

Attempts to log the user in to the Twitter app with the specified appId. Requires a block to handle the case where multiple twitter accounts are registered with the OS. The block will be given an array of usernames and should return the index of the desired account. If, after seeing the list, no account is selected, return NSNotFound.

- (void)loginToTwitterAppWithId:(id)appId multipleAccountsHandler:(id)accountSelection withCompletionBlock:(id)block

Parameters

appId

The Twitter app id to log the user into

accountSelection

A block to handle the case where the OS presents multiple Twitter accounts to choose from. Return the index of the desired username, or NSNotFound

block

A block that will be called with the results of the login attempt.

Discussion

Attempts to log the user in to the Twitter app with the specified appId. Requires a block to handle the case where multiple twitter accounts are registered with the OS. The block will be given an array of usernames and should return the index of the desired account. If, after seeing the list, no account is selected, return NSNotFound.
